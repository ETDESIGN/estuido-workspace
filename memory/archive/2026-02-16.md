## Token Optimization Attempt - Lessons Learned

### What Was Tried
Created a "Strict Tool Scoping" profile system to save tokens by disabling unused tools:
- CODING profile: file-edit + terminal only
- RESEARCH profile: browser + search only  
- MINIMAL profile: no tools

### What Failed
**OpenClaw config validation rejected the profile system:**
- `meta.profile` - unrecognized key
- Root-level `profile` - unrecognized key
- `tools.web.enabled` - unrecognized key
- `tools.media.enabled` - unrecognized key
- `tools.browser` - unrecognized key
- `skills.allowBundled: false` - must be array `[]`, not boolean

### Key Insight
**OpenClaw does NOT support runtime tool profiles.** Configuration is static - no selective tool activation mechanism exists. The tool schemas are loaded at startup and cannot be dynamically filtered per request.

### What Actually Works for Token Savings
The user applied these valid optimizations:
```json
{
  "commands": {
    "native": false,
    "nativeSkills": false
  },
  "skills": {
    "allowBundled": []
  },
  "tools": {
    "elevated": { "enabled": false }
  }
}
```
This disables auto-loading of native commands and bundled skills, saving ~500-1000 tokens/request.

## Token Efficiency Best Practices (Researched)

### OpenClaw-Specific Optimizations
1. **Bootstrap file size limits**
   - `agents.defaults.bootstrapMaxChars: 20000` (default)
   - `agents.defaults.bootstrapTotalMaxChars: 24000` (default)
   - Keep AGENTS.md, SOUL.md concise

2. **Memory management**
   - Use `memory_search` instead of loading full MEMORY.md
   - Daily files (`memory/YYYY-MM-DD.md`) are on-demand
   - Don't auto-load historical context

3. **Skill loading**
   - `commands.native: false` - disable auto tool loading
   - `commands.nativeSkills: false` - disable skill auto-loading
   - `skills.allowBundled: []` - block bundled skills
   - Saves ~1,500 tokens/request

4. **Context compaction**
   - `compaction.mode: "safeguard"` - auto-compact old messages
   - Keeps context window manageable (current: 81K/262K)

### Community Findings
- Typical OpenClaw usage: 5M-50M tokens/month ($10-70)
- Heavy users: 50M-200M tokens/month ($70-150+)
- Our strategy (Kimi + free tiers): ~$25/month = efficient
