## How This Works

1. **Before starting** a task, search memory for relevant workflows
2. **Follow the checklist** — don't skip steps (they're there because we learned the hard way)
3. **After completing** a task, add lessons learned to the relevant workflow
4. **If no workflow exists** for a task type, create one after completing it

## Key Principles

- **Foundation first** — tooling & libraries before features
- **Build after every change** — never let the build break
- **Extract at 2** — when a pattern appears twice, make it shared
- **Document mistakes** — the "what we learned" section is the most valuable part
- **Use stronger models for design** — free models produce inconsistent UI code
